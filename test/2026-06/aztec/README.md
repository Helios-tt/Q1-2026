## Root cause analysis

- **Title**: Escape-hatch proof gate accepted a withdrawal proof that authorized 1158 ETH to the attacker
- **Severity**: `critical`
- **Confidence**: `medium`
- **Violated invariant**: An escape-hatch proof accepted by the verifier must soundly bind public output withdrawals to valid rollup state and entitlement before RollupProcessor updates roots or sends ETH.

### Final root cause

RollupProcessor.escapeHatch accepted attacker-supplied proofData during the escape window and relied on TurboVerifier.verify(bytes,uint256) success for rollup_size 0 as the entitlement gate. The accepted proofData encoded an inner transaction with publicOutput 1158000000000000000000, assetId 0, and outputOwner equal to the attacker EOA, so processDepositsAndWithdrawals called withdraw and transferred ETH. The exact verifier formula, verification-key issue, or proof-system soundness failure that made this proof acceptable is not established by the closed-world artifacts, so this RCA is partial.
