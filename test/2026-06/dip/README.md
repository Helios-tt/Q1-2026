## Root cause analysis

- **Title**: DIP router-exempt transfers execute twice and inflate AMM input accounting
- **Severity**: `critical`
- **Confidence**: `high`
- **Violated invariant**: A single DIP ERC20 transfer call must debit the sender and credit the recipient exactly once for the requested amount; fee exemption must not duplicate the movement.

### Final root cause

DIP._transfer treats transfers involving PancakeRouter as fee-exempt by calling super._transfer(from, to, amount), but the branch does not return and falls through to a second unconditional super._transfer. The attacker routed DIP through PancakePair.skim(PancakeRouter) and PancakeRouter fee-on-transfer swaps so duplicated DIP balance deltas were treated as real AMM input. This let downstream PancakePair swaps satisfy their balance/K checks while outputting excessive AIC, which was swapped to USDC.
